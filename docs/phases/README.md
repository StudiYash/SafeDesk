# Phases

SafeDesk is developed phase by phase so each change can be reviewed for safety, privacy, and public repository hygiene.

Public commits represent reviewed, public-safe progress. They may describe completed milestones at a high level, but they should not publish raw phase reports, Codex prompts, internal execution notes, private implementation plans, or security-sensitive design details.

The public roadmap is available in the root `README.md` as a high-level overview. Detailed execution plans remain local-only so the public repository can showcase SafeDesk without exposing internal build strategy.
